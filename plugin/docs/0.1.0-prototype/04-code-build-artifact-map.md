# Code And Build Artifact Map

Version: `0.1.0-prototype`
Created: 2026-06-24

This file maps the main artifacts created or shaped during the prototype thread.

## Project Areas

| Area | Purpose |
| --- | --- |
| `.codex-plugin/` | Codex plugin manifest/package metadata. |
| `skills/` | Role contracts for phase-controller and builder-team agents. |
| `scripts/` | Local helper tools for `.qc` initialization, recording, validation, and summaries. |
| `assets/qc-templates/` | Seed templates for `.qc` files and phase reports. |
| `docs/` | Architecture, runbooks, schemas, reviews, release planning, and versioned docs. |
| `.qc/` | Local evidence records generated by running the builder-team process. |
| Local audit records | Redacted local Fable5 run records for requested audits. |
| `phase-by-phase-build-log/` | Build log area used during later local build/prototype work. |

## Core Scripts

| Script | Prototype role | Known status |
| --- | --- | --- |
| `scripts/init_qc.py` | Creates `.qc` folders, JSONL files, and templates. | Exists. Does not seed every V04 target record. |
| `scripts/start_phase.py` | Starts a phase run and seeds required markdown files. | Exists. Does not create all target support artifacts. |
| `scripts/record_ponytail_check.py` | Appends Ponytail minimal-code verdict. | Exists. Needs attempt/mode-source hardening. |
| `scripts/record_test_result.py` | Records test result JSONL and appends to test report. | Exists. Needs `required` and `attempt` fields. |
| `scripts/record_deviation.py` | Records deviation JSONL. | Exists. Needs issue/decision/attempt fields. |
| `scripts/summarize_phase.py` | Prints phase summary JSON. | Exists. Does not write summary file. |
| `scripts/validate_phase_record.py` | Validates phase evidence. | Exists. Needs stricter role/test/decision/issue enforcement. |
| `scripts/audit_builder_scope.py` | Records pre-builder baseline and audits post-builder scope changes. | Exists. Needs real-trial validation and hardening. |
| `scripts/builder_team_qc_lib.py` | Shared script helpers and safety scan. | Exists. Safety scan classifies `info`/`warning`/`blocker`; needs real-trial tuning. |
